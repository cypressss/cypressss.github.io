设置代理+burp抓包

发表于 2019-03-23

设置代理+burp抓包

写在前面：

火狐抓包，设置代理都比chrome浏览器要简单，所以一般用火狐。但工作实际测试的时候，需要火狐和谷歌两个浏览器切换，所以还是最好两个都配置好代理，以此记录，节省之后重新开始的时间。平常测试不止是需要抓浏览器的包，有时需要测试APP，还又PC客户端，甚至需要用真机，所以需要在各种环境下配置代理。

记录一下，好记性不如烂笔头。

一、burp破解版安装+常用操作

1.comparer

比较两个数据包

2.右键 change request method

更改请求提交方式，如get->post

3.右键 Engagement tool->Genarate CSRF PoC

生成CSRF PoC 可发送回浏览器，request提交之后burp要记得把请求放掉

4.scanner

抓包，右键 Do an active scan

二、burp抓https的包

火狐浏览器配置代理+导入burp证书抓https包

  1. 安装火狐代理proxyfoxy
    添加127代理（跟burp设置成一样的）
  2. 浏览器访问http://burp 下载证书
    记得先开代理才能打开
  3. 浏览器打开->选项->隐私与安全->证书->导入

  4. 导完证书

    至此，火狐浏览器可以成功抓取https的包

谷歌浏览器配置代理+导入burp证书抓https包

chrome没有自带的代理设置，都是调用系统的，所以导入burp证书也是顺带导入到系统中。

  1. 在应用商店（要翻墙）下载switchtomega（经试验，在网上下载再导入很难成功）
  2. 导入已经下载到本地的burp证书

    ps：另一种方式是直接双击证书（cacert.der）安装
  3. 导入成功
  4. 在给公司电脑导入证书的时候遇到个问题–导不进去。具体是导入“受信任的根证书颁发机构”会自动导进旁边的“中间证书颁发机构”，但是证书并不能生效。
  • 解决办法是直接双击证书，然后导入->勾选存在物理存储区

    这是一个解决办法，具体情况还是要看缘分。:)

三、burp+逍遥模拟器抓包

  1. 官网下载逍遥模拟器（或者夜神）然后设置代理
  2. 代理要设置成本机IP，burp里也一样

    3。 burp设置代理成本机IP
  3. 安装burp证书抓https包
    在安装证书前要设置一个初始密码，不然会安装失败（第一次安装就是因为没有设置初始密码，所以总是失败，关键是还没有提示是哪里有问题，但是这次在自己电脑上还提示了）
  4. 把本地下载好的burp证书拖到模拟器里，双击安装

    没有意外就能顺利安装成功。:)
    ==ps:模拟器里常按鼠标左键显示高级选项==

++先写到这里，祝我下次安装好运，还有看到这篇文章的各位好汉++。

3.winter sleeps

<?php
error_reporting(0);
require __DIR__.'/lib.php';

if(isset($_GET['time'])){

    if(!is_numeric($_GET['time'])){
        echo 'The time must be number.';

    }else if($_GET['time'] < 60 * 60 * 24 * 30 * 2){
        echo 'This time is too short.';

    }else if($_GET['time'] > 60 * 60 * 24 * 30 * 3){
        echo 'This time is too long.';

        sleep((int)$_GET['time']);

    }

题目要求输入时间5184000-7776000之间，这得等死都出不来。可以用科学记数法绕过5.184e6-7.776e6之间都可以过。

4.hard login

要求输入隐藏的用户名和密码，尝试输入admin提示密码错误，说明用户名对了。密码可以尝试弱密码爆破，看别人的wp说是越权，只要抓包改成index.php就可以。

5.URL filtering

上源码

<?php
error_reporting(0);
require __DIR__."/lib.php";

$url = urldecode($_SERVER['REQUEST_URI']);
$url_query = parse_url($url, PHP_URL_QUERY);

$params = explode("&", $url_query);
foreach($params as $param){

    $idx_equal = strpos($param, "=");
    if($idx_equal === false){
        $key = $param;
        $value = "";
    }else{
        $key = substr($param, 0, $idx_equal);
        $value = substr($param, $idx_equal + 1);
    }

    if(strpos($key, "do_you_want_flag") !== false || strpos($value, "yes") !== false){
        die("no hack");
    }
}

if(isset($_GET['do_you_want_flag']) && $_GET['do_you_want_flag'] == "yes"){
    die($flag);
}

highlight_file(__FILE__);

注意到里面有个函数parse_url(),用来解析URL返回其组成部分

源码要求的是返回？之后的数据，
尝试输入http://urlfiltering.solveme.peng.kr/?do_you_want_flag=yes

果不其然返回no hack。

但如果是http://urlfiltering.solveme.peng.kr///?do_you_want_flag=yes
就可以返回flag

因为parse_url()在解析URL时有漏洞，通过 ///x.php?key=value 就可以返回flase

7.array2string

<?php

for ($i = 0; $i < count($value); ++$i) {
    if ($_GET['username']) unset($username);
    if ($value[$i] > 32 && $value[$i] < 127) unset($value);
    else $username .= chr($value[$i]);

    if ($username == '15th_HackingCamp' && md5($password) == md5(file_get_contents('./secret.passwd'))) {
        echo 'Hello '.$username.'!', '<br>', PHP_EOL;
        echo $flag, '<hr>';

highlight_file(__FILE__);

这里要求value的值在ascii码的可见范围之外，而且username是由value拼接成‘15th_HackingCamp’
Chr()自动mod256，所以只要输上述字符的ASCII+256就可以通过。

str="15th_HackingCamp"
for i insstr:
    a=ord(i)
print a+256

密码也不是随便输的，源码提示访问./secret.passwd可以得到密码

11.hell js

F12看到一堆js混淆的代码，直接F12放控制台里跑不出来。
jsfuck有个翻译规则是

eval => []["filter"]["constructor"]( CODE )()

所以说题目源码应该是放在CODE部分里面，所以只要把CODE那部分抠下来放在控制台里跑就可以
复制jsfuck到sublime里方便找CODE

 $res = $sql->query("

1) 输入的name要以a开头

2) name的第二个字母要是d

3) m要放在中间

4) 最后两个字母是in

搭blog的心酸路

发表于 2018-04-22

写在前面：

搭blog真是要累死我，这次是第二次搭了，以前搭的莫名其妙打不开了，气得我先用everthing卸掉以前装的东西的祖宗十八代，然后又注册了个新的github账号重！新！来！装着装着我突然意识到不是本地出问题而是github pages有问题，其实github pages 也没问题，就是我的电脑访问不了，但其他设备都可以正常访问，也是醉了。收拾心情还是挥泪决定记下我心酸的搭建过程。但是基本上只能纯叙述了，因为cmd窗口被我顺手关掉找不到历史记录了。。。。。。

准备阶段

先新建个blog文件夹存放接下来要装的各种东西，原因自然是因为方便找，一定要养成文档分类的习惯，不然迟早坑到自己

安装node.js

第一步我就不得不吐槽了，不要随便装别人博客的东西，因为会造成版本过低，最好是去官网上下一个最新版本的https://nodejs.org/en/
一路安装，安装node.js的时候会顺带装上npm

cmd上敲命令

node -v
npm -v

如果出现版本号就说明安装成功

安装git

也是官网装到刚才新建的blog文件夹https://git-scm.com/

一路默认安装就好

在cmd或者右键git bash 打开敲命令

Git --version

出现版本号说明成功

GitHub的注册和配置

这个按百度出来的步骤做就行

安装配置Hexo

cmd

npm install hexo-cli -g

我看的博客普遍说会报一个warn，但我报了至少三个。。。不管它

查看hexo版本

Hexo version

照样有版本号，也算我安装成功

从这之后就应该在Hexo文件夹里右键git bash敲命令了

新建一个文件夹来初始化Hexo

hexo init cypressss.github.io
cd cypressss.github.io
npm install

本地运行Hexo服务

Hexo s

会提示可以在http://localhost:4000下启动，打开看看如果有Hexo的页面就说明本地配置成功了，千万别按ctrl+c再打开，因为这就关掉了

关联本地和github pages

之前的设置都是只能在本地打开的，那么接下来要让本地的git项目跟远程的github建立联系，用SSH key 实现

Git bash运行

ssh-keygen -t rsa -C "自己的邮箱"

回车会提示输入密码，可以不输直接回车继续

打开 id_rsa.pub 我是用everthing找的，复制里面的内容，粘贴到 github->settings->SSH and GPG key->SSHkeys里

测试下：

ssh -T git@github.com

会有个反馈

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

输yes回车就好

显示successfully表示成功

配置git个人信息

git config --global user.name "自己的名字"
git config --global user.email "自己的邮箱"

在站点配置文件_config.yml->Deployment

1    # Deployment
2    ## Docs: https://hexo.io/docs/deployment.html
3    deploy:
4    type: git
5    repo: git@github.com:cypresssscypressss.github.io.git
6    branch: master

本地文件提交到github：
先安装一个扩展

npm install hexo-deployer-git --save

// 删除旧的 public 文件
hexo clean
// 生成新的 public 文件
hexo g
// 开始部署
hexo d
或者后两条一起写 hexo d -g

在浏览器输入cypressss.github.io 就可以看到hexo 和GitHub pages 成功关联了
